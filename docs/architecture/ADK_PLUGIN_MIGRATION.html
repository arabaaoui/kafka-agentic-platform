ADK Plugin Migration

Mapping custom ↔ natif — Spec 010 ADK 2.x (kafka-agentic-platform)

Spec 010 Migration finalisée — 2026-06-11 google-adk ≥ 2.0

1. Pourquoi cette migration

Situation initiale (ADK 1.33)

  • • PluginChain custom (~120 lignes) appelé manuellement dans BaseAgent._wrap_tool()
  • • _wrap_tool() try/except → string "ERROR: ..." (pas de on_tool_error_callback)
  • • asyncio.gather() avec stagger artificiel 1s pour 3 experts en parallèle
  • • OTelMetricsPlugin stub no-op (observabilité absente)
  • • Monkey-patch api/main.py:5-30 pour ajouter auth_preprocessor manquant
  • • fn.__name__ = "..." pour renommer les tools (pas de Tool Provenance)

Cible (ADK 2.x natif)

  • • BasePlugin.before_tool_callback() / after_tool_callback() natifs
  • • Plugin enregistré sur Runner(plugins=[...])
  • • FunctionTool(fn) avec name depuis fn.__name__, description depuis fn.__doc__
  • • AutoTracingPlugin — OTel gen_ai.client.* en une ligne
  • • MissionContext via session.state["_mission_context"]
  • • Suppression du monkey-patch (fixé upstream en ADK 1.34+)
Non-régression stricte : La migration n'est pas une réécriture fonctionnelle. Chaque mission qui passait sur ADK 1.33 doit produire un résultat sémantiquement équivalent après migration. Les tests golden dans tests/golden/ et le script check_regression.py pilotent la validation.

4. Avant / après — code clé

4.1 Déclaration d'un plugin

Avant — Plugin ABC custom

from core.plugin_base import Plugin

class AutonomyPlugin(Plugin):
    def before_tool_callback(
        self,
        tool_name: str,
        tool_params: dict,
        ctx: MissionContext,
    ) -> None:
        if tool_name in BLOCKED_TOOLS:
            raise AutonomyViolation(tool_name)

    def after_tool_callback(
        self,
        tool_name: str,
        result: Any,
        ctx: MissionContext,
    ) -> Any:
        return result

Après — ADK BasePlugin natif

from google.adk.plugins import BasePlugin
from google.adk.tools import BaseTool
from google.adk.tools.tool_context import ToolContext
from core.plugin_base import get_mission_context_from_state

class AutonomyADKPlugin(BasePlugin):
    def __init__(self) -> None:
        super().__init__(name="autonomy")

    async def before_tool_callback(
        self,
        *,
        tool: BaseTool,
        tool_args: dict,
        tool_context: ToolContext,
    ) -> None:
        ctx = get_mission_context_from_state(
            tool_context.state
        )
        if tool.name in BLOCKED_TOOLS:
            raise AutonomyViolation(tool.name)

4.2 Enregistrement sur le Runner

Avant — PluginChain wrappée dans les tools

# BaseAgent._wrap_tools() retourne des closures
# qui appellent chain.before() / after() autour de fn()
wrapped = [
    self._wrap_tool(fn, mission_ctx)
    for fn in raw_tools
]
agent = LlmAgent(tools=wrapped, ...)
runner = Runner(agent=agent, ...)

Après — plugins= sur le Runner

from google.adk.tools import FunctionTool
from core.plugin_base import _MISSION_CTX_KEY

tools = [FunctionTool(fn) for fn in raw_tools]
agent = LlmAgent(tools=tools, ...)

plugins = build_plugin_list(
    mission_context=mission_ctx,
    tenant_config=self._tenant_config,
)
runner = Runner(agent=agent, plugins=plugins, ...)

# MissionContext dans session.state pour les plugins
await session_service.create_session(
    ...,
    state={_MISSION_CTX_KEY: mission_ctx},
)

4.3 FunctionTool — name + description

Avant — mutation directe __name__

from kafka_agent_toolkit.tools.prom_query import query

query.__name__ = "prom_query"
# Pas de __doc__ → description vide pour ADK
return [query]

Après — __name__ + __doc__ + FunctionTool

from kafka_agent_toolkit.tools.prom_query import query

query.__name__ = "prom_query"
query.__doc__ = (
    "Execute a PromQL query against the environment's "
    "Prometheus instance. Returns the query result as JSON."
)
# Dans base.py (mode natif) :
# tools = [FunctionTool(fn) for fn in raw_tools]
return [query]

5. Migration finalisée

Les feature flags ADK_USE_NATIVE_PLUGINS et ADK_USE_PARALLEL_AGENT ont été introduits comme filets de sécurité lors de la migration ADK 1.33 → 2.x (spec 010). Ils ont été retirés le 2026-06-11 (Phase 8 cleanup) après validation complète.

La plateforme tourne désormais exclusivement sur ADK 2.x natif : build_plugin_list() retourne 9 BasePlugin enregistrés sur Runner(plugins=[...]) ; asyncio.gather() lance les 3 experts simultanément sans stagger artificiel. Aucun code legacy (PluginChain, _wrap_tool, default_plugin_chain) ne subsiste.

6. Workstreams — séquence et statut

FAIT

WS-1 — Bump ADK ≥ 2.0 + suppression monkey-patch

pyproject.toml : google-adk>=2.0,<3. Suppression du bloc api/main.py:5-30 (monkey-patch auth_preprocessor inutile depuis ADK 1.34).

FAIT

WS-2 — PluginChain → ADK BasePlugin

build_plugin_list() construit la liste de 9 BasePlugin pour le Runner. Code legacy (PluginChain, Plugin ABC, _wrap_tool) supprimé en Phase 8.

FAIT

WS-3 — FunctionTool + __doc__ sur tous les tools

agents/kafka_strimzi_expert/agent.py, k8s_gcp_sre/agent.py, prom_alerts_triage/agent.py : fn.__doc__ ajouté sur chaque tool. BaseAgent.run() wrappe systématiquement en FunctionTool(fn).

FAIT

WS-4 — AutoTracingPlugin (OTel natif ADK 2.2)

AutoTracingPlugin() ajouté en position 9 dans build_plugin_list(). Remplace le stub no-op OTelMetricsPlugin (supprimé avec le code legacy en Phase 8). Émet gen_ai.client.operation.duration, gen_ai.client.token.usage.

FAIT

WS-5 — Fan-out sans stagger

orchestrator.py : asyncio.gather() sans délai artificiel — 3 experts concurrent immédiat. Le vrai ParallelAgent ADK reste en WS-6 (BaseAgent → BaseNode, STRETCH). Flag ADK_USE_PARALLEL_AGENT supprimé en Phase 8.

STRETCH

WS-6 — BaseAgent → BaseNode (ADK 2.0 breaking)

Hériter de google.adk.agents.BaseAgent (= BaseNode en 2.0). Réécrire run() en générateur d'events. Activer le vrai ParallelAgent. Supprimer l'interpolation {VAR} manuelle (regex fixée en 2.x). Dépend de WS-2..5 stables en LAB.

STRETCH

WS-7 — PgVectorMemoryService (BaseMemoryService)

Wrapper RAGIndex (core/mem0_bridge.py) en BaseMemoryService. Enregistrer sur le Runner. Supprimer _fetch_kb_context() manuel dans BaseAgent. Dépend de WS-6. Ne pas migrer vers VertexAiRagMemoryService (source de vérité Postgres).

7. Non-goals explicites

  • • NE PAS migrer le RAG pgvector vers VertexAiRagMemoryService — la donnée opérationnelle est dans Postgres avec les missions/tenants.
  • • NE PAS adopter Live API / token streaming — la plateforme est batch-investigation.
  • • NE PAS exposer les agents en serveurs MCP — la plateforme est consumer, pas publisher.
  • • NE PAS adopter A2A / Skill Registry / signed agent cards — single-process, single-author.
  • • NE PAS activer BigQuery Agent Analytics — Postgres + audit.jsonl couvrent le besoin.
  • • NE PAS toucher au bloc GKE token-patching de orchestrator.py:158-193 — orthogonal à ADK.
  • • NE PAS modifier les prompts FR — la politique linguistique (ADR-005) ne doit pas dériver.
